## 需求摘要

调整中央 `scripts/apply_downstreams.py` 的默认行为：下游仓库应用 release 后，不再只停留在本地 `workflow-release-*` worktree 提交；而是在自动审查无阻断问题时，直接提交并触发下游 `session_push_autorelease.sh` 合并到各自默认分支。若 auto-release 被环境条件阻塞，则保留本地 worktree 供恢复。

## 变更说明

- 为 downstream fan-out 提交流程新增 `auto_release_after_review` 分支：在 worktree 中应用当前 release 后，先执行 `git diff --check` 与 `check_release.py --json` 作为自动审查，再写入对应 exec record / commit template。
- 当自动审查通过时，`scripts/apply_downstreams.py` 现在会在下游 worktree 完成本地提交后，直接调用下游 `session_push_autorelease.sh` 把 `workflow-release-*` 分支合并到默认分支，并把 merge 后的主分支 sha 写回 summary。
- 如果 auto-release 因主仓不干净、同步落后或冲突等环境条件失败，summary 会返回 `committed-pending-release` 与 `release_error`，同时保留本地 worktree 供后续恢复，不再把这种半完成状态误记为成功。
- 同步更新中央仓 `AGENTS.md`、`docs/USAGE.md` 的执行说明与维护手册，并发布 release `1.0.25`。
