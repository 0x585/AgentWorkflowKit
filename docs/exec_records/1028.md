## 变更说明

- 新增 `start_exec.sh`、`check_exec_plan.py` 与 `prepare_task_commit.sh`，把“先登记开工计划，再开始正式编码 / 提交”的约束收口到 managed runtime。
- `new_exec.sh` 改为必须接收摘要并原生写入 `## 开工计划`，`new_worktree.sh` 不再隐式创建 exec，而是明确提示下一步运行 `start_exec.sh`。
- `commit-msg` 现在会在原有 `验证结果` / `审查结果` 校验前先强制校验 `## 开工计划`，`pre-commit` 的提示也改为推荐 `prepare_task_commit.sh`。
- 中央 contract、`AGENTS.md`、`docs/USAGE.md`、pending worklist 和下游 managed block 同步切换到新流程，并明确纯文档任务只要走 exec / commit 流也必须补齐 `开工计划`。
- `workflow_kit_lib.py` 新增对旧版下游 `new_exec.sh` 的兼容回退，避免子仓在升级到 `1.0.29` 之前因为旧入口参数不支持而中断 fan-out。
