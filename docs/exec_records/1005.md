## 需求摘要

当前多个项目各自维护了 `PublicWorkRegister` 实现，行为已经出现分叉：

- `AgentTask` 保留了一套基础实现
- `AgentTransitStation` 已经演进出包含 interruption / worktree claim 能力的增强实现
- 中央仓库 `AgentWorkflowKit` 需要把这部分能力统一收口，并通过 workflow release 下发到子项目

本次目标：

- 将下游 `PublicWorkRegisterService` 改为中央仓库统一维护
- 保留 `AgentTransitStation` 现有能力上限，避免“向下兼容”反而丢功能
- 通过新的 workflow release 将统一实现应用到 `AgentTask` 与 `AgentTransitStation`

## 变更说明

- 在 `full_codex_flow` profile 中新增受管源码入口：
  - 下游 `src/main/python/<python_package_name>/tooling/service/public_work_register_service.py`
  - 由中央模板 `templates/full_codex_flow/files/src/main/python/public_work_register_service.py.tmpl` 统一生成
- 将中央仓库自己的 `PublicWorkRegisterService` 升级为下游通用实现：
  - 保留 `claim/release/recommend`
  - 补齐 `interrupt_work_item`、`find_worktree_claim`、`get_register_root`
  - worktree 仍统一归一到主项目级 `PublicWorkRegister/<RepoName>`
- 兼容既有下游差异：
  - 同时兼容旧实现里对 `DEFAULT_REGISTER_ROOT` 和 `DEFAULT_REGISTER_BASE` 的 patch / override
  - `AgentTask` 与 `AgentTransitStation` 的既有测试入口都无需改造即可继续工作
- `workflow_kit_lib.py` 支持对受管输出路径做模板渲染
  - 这样 profile 可以按 `python_package_name` 把同一份中央模板分发到不同包路径
- 发布并应用 `full_codex_flow@1.0.4`
  - 中央仓库自身已更新到 `1.0.4`
  - `AgentTask`、`AgentTransitStation` 已通过中央仓库下发到 `1.0.4`
- 文档更新：
  - 明确下游 `PublicWorkRegisterService` 现在属于中央受管内容，不再建议各子项目分叉维护

## 风险与回滚

- 风险：
  - 下游项目中与 `PublicWorkRegisterService` 强绑定的自定义行为，后续需要继续走中央模板演进，不能再在子项目里单独分叉，否则会在下次 apply release 时被覆盖

- 回滚：
  - 删除 profile 中新增的受管 service 入口
  - 回退中央 service / template / 测试
  - 重新发布一个不包含该文件的 workflow 版本并重新 `apply_downstreams`
