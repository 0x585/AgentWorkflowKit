enforce descriptive branch naming

## 开工计划

- 工作类型：新需求
- 目标：在中央 workflow runtime 中强制执行语义化 `codex/<purpose>` 分支命名，避免 `help`、`pcr` 等模糊或纯缩写分支继续通过官方入口与 code-mode guard。
- 改动范围：`.workflow-kit/` 分支命名源脚本与 guard、共享命名策略 helper、profile managed entries、中央文档与下游 managed block、release 导出与发布产物、对应回归测试。
- 实施步骤：1. 新增共享 branch naming helper 并接入 `new_branch.sh` / `assert_workspace.sh` 2. 更新 profile、文档和 managed block 文案 3. 补充 workflow release 测试并跑导出/发布 4. 完成 review、prepare_task_commit 和 git commit。
- 预期验证：`new_branch.sh --dry-run` 生成可读 slug 且拒绝模糊名；`new_worktree.sh` 继承失败路径；手工切到 `codex/help` 时 `ASSERT_PURPOSE=code ./.workflow-kit/assert_workspace.sh` 失败；`export_templates`、`publish_release` 与新增测试通过。
- 已知风险/阻塞：当前仓已有短分支样例；本轮只在中央 workflow 收口，不直接批量改名历史分支，因此需确保 guard 错误信息足够明确，便于后续 rename/cleanup。

## 变更说明

- 新增共享 branch naming policy helper，统一负责 branch derive / validate；自然语言输入生成可读 slug，模糊单词与单 token 2-4 字符短名直接 hard fail。
- `new_branch.sh` 改为委托 helper，删除旧的首字母缩写回退；`assert_workspace.sh` 在 code mode 下追加 branch 语义校验，阻止手工切到 `codex/help` 这类分支后继续编码。
- 下游 release worktree 创建改为把显式 `codex/workflow-release-*` slug 直接传给 `new_worktree.sh`，避免版本号分支被错误压缩成 `workflow-release-1`。
- 同步更新 contract / AGENTS / USAGE 文案、managed block 模板、release artifacts 与回归测试，发布 `full_codex_flow@1.0.31`。
