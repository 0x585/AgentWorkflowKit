## 需求摘要

推进 workflow 两部分增强：一是为 downstream apply 提供按仓库重试与复用既有 `workflow-release-*` worktree 的恢复入口；二是把中央仓库的“测试 -> 审查 -> 提交”约束补成结构化记录，并把测试/审查结果与最终 staged snapshot 绑定，随后发布新 release `1.0.21`。

## 变更说明

- 为 `scripts/apply_downstreams.py` 增加 `--repo-id` 与 `--resume-existing-worktree`，支持单仓恢复失败 fan-out，并在 `scripts/workflow_kit_lib.py` 里复用既有 downstream worktree / exec record。
- 为 downstream 自动提交注入 `SKIP_POST_COMMIT_AUTOMATION=1`，避免子仓本地修复提交时再次触发整段 `post-commit` 自动化。
- 将 exec record 模板、`commit-msg` 校验和 `prepare_commit.sh` 串起来，要求 `验证结果` / `审查结果` 使用固定字段，并让两个 `提交快照` 与最终 staged snapshot 匹配。
- 新增 `python3 ./.workflow-kit/exec_record_hygiene.py --sync-staged-snapshot --exec-id <exec_id>`，用于在最后 staging 后自动回填快照。
- 更新中央仓库 `AGENTS.md`、`README.md`、`docs/USAGE.md` 和 workflow block 模板，并发布 release `1.0.21`。

## 审查结果

- 审查方式：人工审查 `git diff --stat` 与关键文件 diff，重点检查 downstream worktree 复用、`post-commit` 跳过开关、staged snapshot 递归问题和模板同步结果
- 结论：实现与需求一致；恢复入口只在显式参数下启用，`提交快照` 改为基于排除 `docs/exec_records/**` 的 staged diff hash，避免 exec record 自身导致递归失配；未发现需要额外修正的问题
- 残余风险：若某个恢复 worktree 里同时残留多个 placeholder exec record，当前策略会复用最新一条占位记录；这是恢复场景下可接受的折中，但仍依赖操作者保持 worktree 整洁
