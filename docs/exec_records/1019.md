## 需求摘要

在中央 `AgentWorkflowKit` 中限制 `workflow_guard` 的自动 release apply 边界，避免下游仓库在 primary checkout 或默认分支上因“入口失败 + 版本过旧”而被直接改脏；发布新 release，并将更新应用到 `AgentTask`。

## 变更说明

- 为 `workflow_guard` 增加 primary/default-branch auto-apply 保护：若当前位于 primary checkout 或默认分支，检测到已安装 release 过旧时不再直接执行 `apply_release.py`，而是 fail closed 并输出迁移指引。
- 保留 `codex/*` worktree 内的既有体验：入口失败且 release 过旧时，仍会自动 apply 最新已发布 release 并重试一次。
- 将这条边界同步写入中央仓库维护文档，并发布 workflow release `1.0.19`。
- 为 downstream runtime 模板同步导出新的 `workflow_guard.sh`。
