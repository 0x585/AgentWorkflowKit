## 变更说明

- `session_push_autorelease.sh` 不再因 source branch 相对 target 为 `behind` 或 `diverged` 就提前退出，而是记录同步状态后继续尝试 direct merge；只有真实 merge conflict 才进入既有 conflict-state / resume 流程。
- `pre-commit` warning、workflow contract、中央维护文档与 AGENTS 同步改口：`session_sync.sh` 继续保留，但定位为“想线性化历史时的手动工具”，不再暗示它是 auto-release 的必经前置步骤。
- 通过 `scripts/export_templates.py` 回写模板，并发布新的 `full_codex_flow` release `1.0.32`，让下游仓库能接收到这次 autorelease 策略变更。
- `tests.test_workflow_release` 新增/更新 direct-merge 和 conflict 场景，覆盖 behind/diverged clean merge、真实 merge conflict，以及 pre-commit warning 文案。

## 风险与回滚

- 风险：这次改动会放宽 autorelease 的前置策略，如果某些维护者之前依赖“先 rebase 再 merge”的隐式流程，新的行为会更偏向直接 merge；真实冲突仍会停下，但 merge 入口会更频繁地被执行。
- 回滚：回退本次中央源码、模板与 `1.0.32` release 提交，并重新发布上一版 release；下游若已应用，可再运行上一版 release 的 apply 进行覆盖恢复。
